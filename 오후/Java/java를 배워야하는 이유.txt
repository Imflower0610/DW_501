저희 조는 java spring을 배우는 것이 좋다고 의견을 모았습니다.
저희 조가 java spring을 선택한 이유는 크게 3가지로 나눌수 있는데요,
바로 실무에 유리하다, 취업에 유리하다, 전망에 유리하다는 세가지 입니다.

1. 실무에 유리하다.
 nodejs는 java script를 베이스로 만든 언어라서 상당부분 비슷하기 때문에 java에 비해 진입 장벽이 낮고, 복잡한 과정도 없어 비교적 쉬운 언어이고,
java는 네트워크 기술에 대한 구조의 이해가 선행되어야하고 코드작성에도 난이도가 있습니다.
 따라서 node.js가 java보다 배우기에 쉬울 수 있습니다.
하지만 실무에서 node.js만 사용하는 경우는 극히 드물고 java를 함께 사용하는 경우가 대부분입니다.
만약 지금 node.js를 java 대신 배운다면 실무에서 java를 처음 접하게 될 것이고, 
앞서 말했듯이 java가 node.js보다 접근하기 어렵기 때문에 차라리 지금 java를 접해서 조금 더 익숙해지는 것이 더 낫다고 판단했습니다.

2. 취업에 유리하다.
 구인구직사이트에 검색시 공기업의 경우에는 node.js기술자를 구하는 기업이 하나도 없었고, 사기업의 경우에도 java사용 가능자를 구하는 비율이 월등히 높았습니다.
node.js가 spring에 비해 훨씬 쉽고 구현하기 편하여 중소기업들이나 스타트업에서 거의 필수로 사용하고 있는 상황이지만
아직까지는 국내에 spring을 요구하는 기업이 더 많기도 하고 대기업은 대부분 spring체제도 돌아가고 있기 때문에 java를 배워두는 것이 취업에 더 유리하다고 볼 수 있습니다.

3.전망에 유리하다.
node.js는 현재 java의 대체 언어로 평가받고 있지만 나온지 10년이 되지 않았고 node.js만 단독으로 사용하는 기업은 중소기업이나 스타트업 뿐이며, 이 역시도 소수입니다.
물론, 콘텐츠 산업이 점점 부흥하는 추세인 만큼 영상과 이미지를 많이 다루는 사이트들에서는 node.js를 사용하는 것이 더 유리할 수 있습니다.
그러나 자바는 현재 공기업과 대기업, 연구시설, 금융기관 등에서 많이 사용하고 있고, 전자정부 프레임워크(정부)는 자바로 작성된 케이스가 훨씬 많습니다.
즉, 새로 시작하는 기업이 아니라 현재 개발된 서비스의 대부분은 java기반이므로 이를 유지보수할 때 java기술자들은 계속 필요하며 그 안정성과 보안성에 있어 java를 찾는 추세입니다.
같은 이유로, 기존의 서비스가 아닌 신규서비스를 개발할 때도 아직까진 java가 우세하기 때문에, 전망에 있어서도 아직까진 node.js가 java를 대체할 수 있는 언어로 볼수는 없습니다.

이와 같은 측면에서 바라볼때, java를 우선해서 배우고 싶다는 것이 저희 팀이 의견입니다.
